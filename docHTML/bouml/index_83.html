SymbolTableclass
SymbolTableartifactIn order to manage scopes, the VSL symbol table is implemented as a stack
of tables, each of them being a map type data structure
(from string identifiers to symbols (Operand3a).

Every insertion of a pair (key, value) is done in the table which is on top of stack.

The lookup method proceeds first with the top of stack, and if nothing is found there,
it recursively descends the stack. It ends the search at the first occurrence of the key or
when it reaches the bottom of the stack.

@author MLB
SymbolTableoperationCreates a new symbol table. The current scope is zero.
